Your first module

A Dunk module is one node in the tree of a Dunk store. If you've used Redux Toolkit, it's like a slice, but for a tree rather than a flat organization.

At a high level, a Dunk module has a few core elements:

  • Local state: the data stored in the module (not including the state of its children)
  • Actions: the types of actions that the module can handle
  • Reducers: the functions that handle the module's actions
  • Children: the module's children

Let's begin by writing the root module of the store.

Defining the type of the root moduleâ€‹

Dunk provides a utility type, DunkModule, to help you easily define the type of a module. It takes three arguments: local state, payloads, and children.

State typeâ€‹

Let's first define the type of the module's state:

type RootState = {
  counter: number;
}

Children typeâ€‹

Every Dunk module has a children type, which is a map of names to child DunkModules.

For now, our module has no children, so we'll define a trivial type:

type RootChildren = {}

Payloads typeâ€‹

Dunk modules accepts a payloads type, which is an object that maps action names to payload types.

For example, let's say we want an action called setCounter that accepts a number as a payload. We can define our payloads type like this:

type RootPayloads = {
  setCounter: number;
}

Let's add two more actions, reset and increment, which don't need a payload. For this case, their payload type can just be undefined:

type RootPayloads = {
  setCounter: number;
  reset: undefined;
  increment: undefined;
}

Putting these types togetherâ€‹

To define the type of a module, just use the DunkModule utility type and pass in the state, children, and payload type, in that order:

export type RootModule = DunkModule<RootState, RootChildren, RootPayloads>;

Writing the moduleâ€‹

Now that we've defined the type of our root module, we can use the createDunkModule function to actually write the module.

Begin by calling the function and passing in the RootModule type as the generic type parameter:

export const rootModule = createDunkModule<RootModule>(
  // ...
);

createDunkModule accepts three arguments.

Argument 1: Initial stateâ€‹

Pass in an object that satisfies the module's state type â€” in this case, RootState.

export const rootModule = createDunkModule<RootModule>(
  {
    counter: 0
  },
  // ...
)

Argument 2: Childrenâ€‹

Pass in an object that satisfies the module's children type â€” in this case, RootChildren.

For now, since the root module has no children, we can just pass in an empty object:

export const rootModule = createDunkModule<RootModule>(
  // ...
  {},
  // ...
)

Argument 3: Reducersâ€‹

For the final argument, pass in a map of action names to reducers that can handle those actions.

A reducer is a function that accepts current state and the action's payload, and returns an updated state.

Thanks to the module's payloads type (RootPayloads in this case), the following things happen automatically:

  • Dunk ensures that we have reducers for all of the actions we've defined
  • Dunk automatically types the state, payload, and return type of each reducer

Let's now write our three reducers:

export const rootModule = createDunkModule<RootModule>(
  // ...
  {
    setCounter: (state, payload) => ({
      ...state,
      counter: payload,
    }),
    reset: (state) => ({
      ...state,
      counter: 0,
    }),
    increment: (state) => ({
      ...state,
      counter: state.counter + 1,
    }),
  }
)

Notice that our reset and increment reducers don't accept a payload argument since their payload types are undefined.
