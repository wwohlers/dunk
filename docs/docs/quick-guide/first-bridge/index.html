Your first bridge

Each module has a corresponding bridge, which is an object specifically for interacting with that module.

At a high level, a bridge has a few core elements:

  • Action creators: functions that return actions for the module
  • Selector creators: functions that return selectors for the module
  • Extras: an arbitrary object that can dispatch the bridge's actions and use its selectors
  • Children: the bridge's children (analogous to the module's children)

Let's now write the bridge for the root module.

createDunkBridgeâ€‹

To use a Dunk bridge, we'll use the helper function called createDunkBridge:

export const rootBridge = createDunkBridge(
  // ...
)

This function takes 5 arguments.

Argument 1: The dunk moduleâ€‹

First, pass in the module whose bridge you're creating:

export const rootBridge = createDunkBridge(
  rootModule,
  // ...
)

Argument 2: action creatorsâ€‹

Action creators are functions that accept any arguments and return actions that can be handled by the module. Dunk will type check your action creators to make

Often, but not always, your action creators will map 1-to-1 with your reducers, like this:

export const rootBridge = createDunkBridge(
  // ...
  {
    setCounter: (value: number) => ({
      type: "setCounter",
      payload: value,
    }),
    reset: () => ({
      type: "reset",
    }),
    increment: () => ({
      type: "increment",
    }),
  },
  // ...
)

Argument 3: selector creatorsâ€‹

A selector is a function that accepts a module's state and returns any arbitrary value.

createDunkBridge's 3rd argument is an object whose values are selector creators, i.e., functions that return selectors for the module.

Most of your selector creators will likely have no arguments. Also, often, your selector creators will map 1-to-1 with your module's state, like this:

export const rootBridge = createDunkBridge(
  // ...
  {
    counter: () => (state) => state.counter,
  },
  // ...
)

Argument 4: extrasâ€‹

Extras is a function that takes in an object with four keys:

  • dispatch: function to dispatch an action
  • select: function to get the value returned by a selector
  • actionCreators: the action creators object passed as the second argument
  • selectorCreators: the selector creators object passed as the third argument

Extras can return anything. It's a good place to define more complex logic, including any asynchronous logic. For example, you can use extras to retrieve data from an API and dispatch it to your store.

Let's say we want functionality to pull a new value for our counter from an API, but only if the counter's value is 0. Here's how we can use extras to achieve this:

export const rootBridge = createDunkBridge(
  // ...
  ({ dispatch, select, actionCreators, selectorCreators }) => {
    return {
      retrieveCounter: async () => {
        const currentCounterValue = select(selectorCreators.counter());
        if (currentCounterValue > 0) {
          const newValue = await Api.fetchCounterValue();
          dispatch(actionCreators.setCounter(newValue));
        }
      }
    }
  }
  // ...
)

Argument 5: childrenâ€‹

Finally, you'll need to pass in the bridges of the module's children. Since the module has no children yet, we can just pass in an empty object:

export const rootBridge = createDunkBridge(
  // ...,
  {},
)
